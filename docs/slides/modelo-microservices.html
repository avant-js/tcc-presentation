MODELO


  • Node-RED como base para modelagem de aplicações
  • Cada aplicação desenvolvida será um serviço separado
  • Lógica no serviço e comunicação por REST
  • Possibilidade de escolha das bibliotecas utilizadas
  • Cada serviço terá sua própria configuração de persistência
  • Integração com ferramentas de entrega contínua
  • Possibilidade de atualização da aplicação
  • Uso de módulos Node.js populares e funcionalidades nativas do JavaScript
  • Uso do Yeoman para geração de código
  • Integração com Swagger para documentação
